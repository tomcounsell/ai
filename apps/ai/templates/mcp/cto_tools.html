CTO Tools provides structured frameworks and best practices for common engineering leadership activities. Helps AI assistants guide technical leaders through weekly reviews, team analysis, and strategic decision-making with analysis-focused, non-prescriptive approaches.

INSTALLATION

RECOMMENDED

One-Click Installation

Download the MCP Bundle for instant installation. Works with Claude Desktop, Cursor, Windsurf, and other MCP-compatible clients.

📦 DOWNLOAD CTO-TOOLS.MCPB
After downloading: Double-click the file to install. Your MCP client will handle the rest. No configuration needed.

⚙️ Advanced: Manual Installation

Prefer to configure manually? Expand your client below for detailed instructions.

Claude Desktop

Edit: ~/Library/Application Support/Claude/claude_desktop_config.json 

{
  "mcpServers": {
    "cto-tools": {
      "url": "https://ai.yuda.me/mcp/cto-tools/serve"
    }
  }
}

After saving, restart Claude Desktop (Cmd+Q then reopen). Look for the 🔌 icon.

Claude Code (CLI)

Run this command in your terminal: 

claude mcp add cto-tools --url https://ai.yuda.me/mcp/cto-tools/serve

AVAILABLE TOOLS

TOOL_01

weekly_review()

Streamlined 3-phase framework for weekly engineering team reviews. Analysis-focused, not prescriptive. Works with ANY codebase.

Phase 1: Data Gathering

  • Parallel git commands for efficient data collection
  • Commit analysis across the reporting period
  • Contributor activity tracking

Phase 2: Internal Analysis

  • Review commits and categorize work
  • Identify patterns and themes
  • Generate contributor statistics

Phase 3: Concise Output

  • ~200 word summary with 5 adaptive categories
  • Contributor recognition and stats
  • Communication-ready format for any channel
Characteristic Implementation
Review Time 15-20 minutes
Output Length ~200 words
Categories 5 adaptive, based on actual work
Codebase Universal, no configuration needed
