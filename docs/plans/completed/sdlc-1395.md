# SDLC Health: Stalled Pipeline Detection and Skills-Audit Issue Filing

Closes #1395 (consolidated from #1250 and #1299).

## Problem

Two SDLC health gaps allow problems to accumulate silently:

**Gap 1 — State-level SDLC idle (stalled PRs):**
The `session-liveness-check` reflection (`agent/session_health.py`, runs every 5 min) detects wedged *processes*, not stalled *state machines*. A pipeline can reach a state where: the PM session completed, no dev session is running, but the PR is open and the plan shows an incomplete stage. Nothing fires. The pipeline sits idle indefinitely until a human notices.

Real example: PR #1237 (issue #1197) and PR #1236 (issue #1218) sat with commits ~8 hours old while their PM sessions had gone dormant. No reflection was watching for this state-level stall. The CEO had to ask "did anything happen overnight?" to surface them.

**Gap 2 — Skills-audit failures not filed as issues:**
The `skills-audit` reflection (`reflections/auditing.py:516`) runs nightly and checks each skill's SKILL.md for structural problems (missing required sections, broken tool references, etc.). Currently it only logs findings — there is no feedback loop into the issue tracker. FAIL findings silently accumulate in the reflection record and worker log, going unactioned.

**Desired outcome:**
1. A new reflection `sdlc-progress-check` (every 30 min) detects SDLC pipelines that are idle at the state level and sends a Telegram alert after a configurable idle threshold (default: 4 hours).
2. The `skills-audit` reflection files a deduplicated GitHub issue for each FAIL finding, gated by two consecutive FAIL runs to filter transient audit-script regressions.

## Data Flow

### Phase 1 — Stalled Pipeline Detection

1. **Entry point**: `ReflectionScheduler` fires `sdlc-progress-check` every 1800s (30 min).
2. **`run_sdlc_progress_check()`**: Calls `run_per_project_audit(_check_project_stalls, name="sdlc-progress-check")`.
3. **`_check_project_stalls(project)`** per project:
   - Calls `gh pr list --state open --json number,headRefName,isDraft,baseRefName` (cwd=project working dir) to get open PRs with SDLC branches (`session/sdlc-*`).
   - For each SDLC PR, checks: not draft, issue still open (`gh issue view N --json state`), last commit age via `git log --format=%ct -1 origin/session/sdlc-{N}`, no active AgentSession (`AgentSession.query.filter(slug=slug, status__in=NON_TERMINAL_STATUSES)`).
   - If all stall gates pass, checks Redis dedup key `sdlc:stall:alert:{slug}:{last_commit_sha}` (TTL = cooldown × 3600). If absent: emit Telegram alert and set the key.
4. **Alert**: `subprocess.run(["valor-telegram", "send", "--chat", "Dev: Valor", message])`.
5. **Return**: aggregated findings dict.

### Phase 2 — Skills-Audit Issue Filing

1. **Entry point**: Reflection scheduler invokes `run_skills_audit()` on its cron tick.
2. **Per-project iteration**: `run_per_project_audit` walks every project, calls `_skills_audit_for_project(project)`.
3. **Per-project audit**: subprocess invokes `audit_skills.py --no-sync --json`, parses stdout, extracts FAIL findings.
4. **NEW — streak update**: For each FAIL finding `(skill, message)`, compute SHA-256 hash of `f"{project_slug}/{skill_name}/{rule_id}"`. Increment `skills_audit:streak:{hash}` in Redis with 7-day TTL.
5. **NEW — gate**: If streak >= 2 AND `skills_audit:issues_filed:{hash}` does not exist: acquire per-finding lock (`SET NX EX 60`), fire `gh issue create` against the project's repo. On success, set 30-day dedup key.
6. **Aggregation**: per-project `findings` + `issues_filed` count flow up into the reflection record summary.

## Failure Path Test Strategy

### Phase 1 Exception Handling
- `gh pr list` subprocess failure: log WARNING, return empty findings. Test: mock `subprocess.run` to raise `FileNotFoundError` → no exception propagates.
- `git log` non-zero exit (branch not found): log DEBUG, skip slug. Test: mock returncode != 0 → slug skipped, no alert.
- `AgentSession.query.filter` raises (Redis down): catch, log WARNING, return `{status: "error"}`.
- `valor-telegram send` failure: log WARNING, finding still logged. Test: mock FileNotFoundError → no exception.
- Redis unavailable for dedup write: skip Telegram send rather than potentially spamming.

### Phase 2 Exception Handling
- No new `except Exception: pass` blocks. Each catch arm logs a warning.
- `gh` CLI failure (returncode != 0): do NOT set dedup key — retry on next run.
- Redis outage: issue-filing degrades to "fire every run" but telemetry continues uninterrupted.
- Empty findings list: 0 issues filed, 0 Redis writes. Test: assert no-op.
- `gh repo view` failure: log warning, skip issue filing for that project's findings only.

### Race Conditions
- **Phase 1**: Stall check fires during the <30s window after dev session goes terminal but before PM session wakes. The 4h threshold makes this impossible to trigger — far longer than PM session wake-up latency (~seconds). No locking needed.
- **Phase 2**: Two concurrent reflection ticks process the same finding. Mitigation: `SET NX EX 60` filing lock before `gh issue create`. Only the first writer proceeds; the second sees the lock and skips.

## Rabbit Holes

- **Phase 1**: Do NOT implement auto-PM-session-restart. Notification-only first — calibrate alert noise before adding recovery automation.
- **Phase 1**: Do NOT extend `session-liveness-check` directly (runs every 5 min, would call `gh pr list` 12×/hr — wasteful). A separate 30-min reflection is cleaner.
- **Phase 2**: Do NOT restructure `_file_issue_if_new` in `docs_auditor.py` into a shared helper — premature extraction; revisit if a third caller appears.
- **Phase 2**: Do NOT add a `--file-issues` flag to `audit_skills.py`. The streak gate is a reflection-cadence concept that doesn't make sense for one-off CLI invocations. Keep the script side-effect-free.
- **Both phases**: Raw Redis dedup namespaces (`sdlc:stall:alert:*`, `skills_audit:*`) are NOT Popoto-managed model keys. The CLAUDE.md "no raw Redis on Popoto-managed keys" rule does not apply. These are pure bookkeeping keys matching the established precedent in `docs_auditor.py:112` and `agent/sustainability.py:97-98`.
