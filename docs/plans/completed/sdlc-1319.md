# Add 6 New Skills: ontologies, grill-me, deepen, observability, zoom-out, tdd

## Problem

The skill library covers the SDLC pipeline and system operations but has no skills for:
1. **Domain vocabulary management** — no `ONTOLOGIES.md` convention, no skill to build one
2. **Proactive design auditing** — no skill to flag shallow modules before they become tech debt
3. **Module instrumentation discipline** — no written standard for making new modules debuggable
4. **Human interrogation** — every skill examines artifacts; none examine the human's understanding
5. **In-session course correction** — no "am I solving the right problem?" check mid-session
6. **TDD sequencing** — no skill enforces the red→green→refactor loop or reusable test-quality rubric

**Current behavior:** Agents and users have no `/skill-name` command for any of the above six capabilities. The gaps are non-overlapping with existing skills.

**Desired outcome:** Six new global skills in `.claude/skills-global/`, each with a `SKILL.md` following the canonical frontmatter convention, synced to `~/.claude/skills/` on the next `/update` run. One shared artifact (`_shared/test-quality.md`) in `.claude/skills/` for project-level reference.

### ONTOLOGIES.md Format Convention

The `ontologies` skill maintains an `ONTOLOGIES.md` at the repo root with this format:

```markdown
# Ontologies — <Repo Name>

> Canonical domain vocabulary for this repo. Updated by `/ontologies`.
> Place next to CLAUDE.md. Terms are domain concepts, not code identifiers.

## <Domain Area>

### <Term>
**Definition:** One precise sentence.
**Usage example:** "When a user cancels an Order, the…"
**Contrast with:** (optional) other terms this is commonly confused with
```

## Risks

### Risk 1: Frontmatter field mismatch with Claude Code's current spec
**Impact:** Skill doesn't appear in the available-skills list or triggers incorrectly
**Mitigation:** Copy frontmatter structure directly from an existing working global skill (e.g., `do-build/SKILL.md` or `analyze/SKILL.md`) and replace content

### Risk 2: `_shared/test-quality.md` reference in do-test/do-patch breaks existing invocations
**Impact:** do-test or do-patch fails when attempting to load the sub-file
**Mitigation:** The reference is a "When to load sub-files" entry — Claude reads the file if present; if the path is wrong, it skips. Add the reference with a conditional ("if running on a new module") to avoid forced reads

### Risk 3: `ontologies` delegates to `grill-me` but grill-me's interface is implicit
**Impact:** ontologies invokes grill-me incorrectly (wrong trigger phrase, wrong argument passing)
**Mitigation:** In the ontologies SKILL.md body, instruct the agent to invoke grill-me by telling the user "running /grill-me on this topic" — the delegation is behavioral, not a code call
