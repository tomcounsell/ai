# Remove dead `behavioral_learning` reflection

## Problem

`reflections/behavioral_learning.py` runs daily, exits in `0.0s`, and logs:

```
[behavioral_learning] models.cyclic_episode not available — skipping episode cycle-close and pattern crystallization
```

The producers it depends on (`CyclicEpisode`, `ProceduralPattern`,
`AgentSession.tool_sequence`, `AgentSession.friction_events`) were never ported
to `main`. The reflection was extracted from the reflections monolith in PR
#967 along with its consumer surface, but the producer half stayed on a closed,
unmerged branch. Result: the daily slot occupies a runtime row, fires every
86400s, and produces nothing.

This is precisely the "skip-with-status-`ok`" false-positive shape that the
hourly `agent-session-cleanup` reflection (#1271) was built to surface and
prune. It does not crash — it logs, returns `{"status": "ok", ...}`, and
disappears — which is the worst kind of dead reflection.

**Current behavior:**
- `behavioral-learning` slot is `enabled: true` with `every: 86400s` in
  `~/Desktop/Valor/reflections.yaml` (symlinked from `config/reflections.yaml`).
- Each fire takes the `ImportError` branch in `reflections/behavioral_learning.py`
  and returns `summary="behavioral_learning skipped: models.cyclic_episode not installed"`.
- Dashboard groups the slot into `GROUP_HOUSEKEEPING` (`ui/data/reflections.py:50`).
- `docs/features/reflections.md:139` describes the slot as if it does the work.

**Desired outcome:**
- The slot is gone: deleted from yaml, deleted from `reflections/`, deleted from
  the dashboard groupings, deleted from feature docs.
- After the change, `tail -n 100 logs/worker.log` over one reflection cycle does
  not contain `models.cyclic_episode not available`.
- The trajectory feature is documented as deferred pending the upstream Popoto
  `TrajectoryMemory` recipe (`tomcounsell/popoto#389`).

## Risks

### Risk 1: iCloud sync race on the yaml edit
**Impact:** A second machine could re-introduce the `behavioral-learning`
block from a stale local copy of the vault if it modifies the same file
during the sync window.
**Mitigation:** The yaml is owned by a single editor at a time in practice
(no concurrent edits). The change is committed to the repo via the symlink
under `config/reflections.yaml`, so any conflict is caught at git-commit time
on the second machine. After merge and `/update` on the other machine,
iCloud sync converges on the post-PR state.

### Risk 2: Live worker still has the old yaml in memory
**Impact:** A worker process that booted before the yaml edit has the
`behavioral-learning` slot in its registry. It will continue firing the slot
until restarted, hitting an `ImportError` on the now-deleted module rather
than the previous `models.cyclic_episode` `ImportError`.
**Mitigation:** Restart the worker after merge per the standard
`./scripts/valor-service.sh restart` workflow (project rule #10). The
`agent-session-cleanup` hourly reflection (#1271) would also surface the new
import error within an hour. Since the dotted callable is removed, the
scheduler will log a registration error and skip the slot at the next config
reload — not crash.

## Race Conditions

No race conditions identified — all changes are file-level edits committed in
one PR, applied atomically at git-commit time. The runtime impact is "the
next scheduler boot reads the new yaml"; there is no shared in-memory state
mutation across actors.
