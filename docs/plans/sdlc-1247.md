# Consolidate docs hygiene: unified auditor substrate replacing 5 disjointed pieces

## Problem

Documentation hygiene is spread across five disjointed pieces with no shared logic, no shared
output format, and inconsistent safety properties. Each one solves part of the problem; together
they create maintenance burden and seesaw risk.

**Current behavior:**

| # | Piece | Type | Scope | Output | Status |
|---|-------|------|-------|--------|--------|
| 1 | `feature-docs-audit` (`reflections/auditing.py`) | reflection (daily) | `docs/features/*.md` | findings → log | active |
| 2 | `documentation-audit` (`reflections/auditing.py`) | reflection (daily) | `docs/*` (excl. plans) | commits to main | active |
| 3 | `knowledge-reindex` (`reflections/memory_management.py`) | reflection (daily) | `~/work-vault/` | KnowledgeDocument upserts | **broken** |
| 4 | `/do-xref-audit` (`.claude/skills/do-xref-audit/SKILL.md`) | skill (manual) | vault ↔ docs | commits to current branch | active |
| 5 | `/do-docs-audit` (`.claude/skills/do-docs-audit/SKILL.md`) | skill (manual) | `docs/*` | opens PR | active |

**Desired outcome:** One auditor substrate (`reflections/docs_auditor.py`) used by two callers:
a new `docs-auditor` rotation reflection, and a refactored `/do-docs` SDLC stage.

## Data Flow

### Caller A — daily rotation reflection

1. **Entry:** Reflection scheduler fires `docs-auditor` callable
2. **Auth probe:** Anthropic (required) + embedding (optional, graceful degradation to lexical-only)
3. **Lock:** `r.set("docs_audit:running:global", "1", nx=True, ex=3600)` — skip if already locked
4. **Dirty-tree guard:** `git status --porcelain` must return empty string. If dirty, release
   lock and return `{"status": "skipped", "reason": "dirty_tree"}` — never check out a new branch
   over uncommitted changes. (No `git stash` shenanigans; the worker should never have a dirty
   tree under steady-state operation, so this is a true guard, not a workaround.)
5. **Rotation:** Read the single Redis hash `docs_audit:last_run` (HGETALL) — see the rotation
   key Implementation Note below — and pick the primary doc with the oldest/missing timestamp
6. **Neighborhood:** Expand from primary doc via outbound links, inbound refs, code symbol paths. Cap at 20 files.
7. **Detectors:** Run auto-fix then file-as-issue detectors
8. **Apply:** Git-apply auto-fixes to working tree
9. **Zero-diff gate:** `git diff --quiet` — if no diff, write `docs_audit:last_completed_run_*`
   liveness keys, update rotation hash, release lock, return
10. **Branch + push:** `git checkout -b docs-audit/{slug}-{ts}` → commit → push
11. **PR:** `gh pr create` (non-draft) with diff stats
12. **Telegram:** `valor-telegram send --chat "Dev: Valor" "..."` (best-effort, swallowed)
13. **Rotation update:** `r.hset("docs_audit:last_run", path_slug, ts)` for all touched primary
    docs and vault docs (vault entries use `vault:{project_key}:{path_slug}` field name in the
    same hash — see Vault Rotation Note)
14. **Liveness signal:** `r.set("docs_audit:last_completed_run_ts", ts)` and
    `r.set("docs_audit:last_completed_run_summary", json.dumps({...}))`
15. **Lock release:** `try/finally` `r.delete("docs_audit:running:global")`

**Implementation Note (per-file Redis key explosion):** Rotation state is stored as a **single
Redis hash** `docs_audit:last_run` with fields keyed by path slug, not as one key per file.
Reads use `HGETALL` (single round-trip); writes use `HSET`. A repo with 200 docs produces 200
hash fields under one key, not 200 top-level keys. This keeps `KEYS docs_audit:*` clean and
makes `redis-cli HGETALL docs_audit:last_run` a single-command audit.

**Implementation Note (vault rotation in Data Flow — addresses missing step):** Vault docs
(per-project) use a separate hash field-prefix within the same hash: `vault:{project_key}:{path_slug}`.
The rotation pick phase reads both top-level and vault-prefixed fields and picks the oldest
across both pools, weighted by `vault_weight` parameter (default 0.5 — vault docs are picked at
half the rate of repo docs, since vault is read-mostly). Step 13 writes vault-prefixed fields
when a vault doc was touched; otherwise only repo fields are updated.

### Caller B — `/do-docs` SDLC stage

1. **Entry:** `/do-docs` skill invoked with PR context
2. **Sub-agents A–D run:** Doc-discovery, as before
3. **Substrate call:** `audit(primary_path=None, scope_mode="pr-changed-files", apply_mode="apply", project_key=...)`
4. **Substrate:** Neighborhood = PR-changed files only (no transitive fanout). Detectors run. Fixes applied.
5. **Commit:** Substrate commits to **current branch** (not a new branch)
6. **Wrapper:** `/do-docs` writes `sdlc-tool stage-marker --stage DOCS --status completed` and plan frontmatter

**Implementation Note (S1/S3 — substrate invocation mechanics):** `/do-docs` is a markdown skill;
it cannot import Python directly. The skill's bash block invokes the substrate via a one-shot
Python module entrypoint:

```bash
python -c "from reflections.docs_auditor import audit; import json, sys; \
  result = audit(primary_path=None, scope_mode='pr-changed-files', \
                 apply_mode='apply', project_key='${PROJECT_KEY}'); \
  print(json.dumps(result))"
```

The substrate exposes `audit()` as a synchronous function (matching existing reflection callable
shape — see `run_documentation_audit` for precedent). Return value is a dict with keys
`{"status", "files_touched", "fixes_applied", "issues_filed"}`. The skill parses the JSON, fails
the stage if `status == "error"`, otherwise proceeds to the stage marker + frontmatter write.

### Caller A — invocation mechanics

The reflection scheduler discovers `audit()` via `config/reflections.yaml`'s `callable:` field
(same mechanism as `run_documentation_audit`). The yaml entry references
`reflections.docs_auditor:run_docs_auditor` — a thin wrapper around `audit()` that handles
rotation-key selection (least-recently-run primary doc) and writes the liveness signal keys.
`audit()` itself is also importable directly from Python for tests and the `/do-docs` caller.

## Solution

### Key Elements

- **Shared substrate** (`reflections/docs_auditor.py`): New module exporting `audit()` callable.
  Absorbs logic from `scripts/docs_auditor.py` (Anthropic analysis), `run_feature_docs_audit`
  (stale-term detection), and `/do-docs-audit` (auto-fix behavior).

- **Auto-fix detectors (safe transforms only):**
  1. Renamed internal markdown links (via `git log --follow --diff-filter=R`)
  2. Renamed file paths / Python symbols in doc body (same `git log --follow` approach)
  3. README index entries pointing at deleted files
  4. Stale-term dictionary mappings (dict at module level: `STALE_TERMS: dict[str, str]`)

- **File-as-issue detectors (judgment required):**
  1. Deleted (not renamed) targets
  2. Stub docs (< 5 content lines)
  3. Overlapping docs / reorganization candidates
  4. Orphan plan files in `docs/plans/` (no linked tracking issue)
  5. Mtime drift signal (informational, filed as priority signal)

  **Implementation Note (SI2 — minimum useful detector set):** The four auto-fix detectors are
  retained as listed (each maps directly to a known existing failure mode in
  `run_feature_docs_audit` / `DocsAuditor`). The five file-as-issue detectors are the **maximum**
  considered set; if any of (3) Overlapping docs or (5) Mtime drift produce more than
  ~1 issue/week of low-signal output during Phase 2, they are deferred (commented out, not
  deleted) before Phase 3 promotion. Decision criterion: < 50% of issues filed by that detector
  during Phase 2 result in human action within 7 days → defer. This is tracked manually via the
  Phase 3 gate review; no automated metric needed.

- **`docs-auditor` reflection:** New daily reflection in `config/reflections.yaml`. Picks primary
  doc via least-recently-audited rotation. Pushes `docs-audit/{slug}-{ts}` branch as a non-draft PR.
  Posts Telegram notification with PR URL + diff stats.

- **`/do-docs` thin caller:** Preserves sub-agents A–D (discovery aids). Adds ~10 lines calling
  substrate with `scope_mode="pr-changed-files"`. Stage markers and plan frontmatter stay in wrapper.

- **`do-docs-branch-sweeper` reflection:** New daily callable. Deletes `docs-audit/*` remote branches
  older than 7 days with no open PR. Closes open `docs-audit/*` PRs older than 14 days.

  **Implementation Note (SI1 — sweeper scope is intentionally narrow):** The sweeper handles
  ONLY `docs-audit/*` branches and PRs originating from `docs-auditor`. It does NOT touch:
  any other prefix, branches without an associated PR-history record, branches with commits from
  authors other than the reflection's commit identity, or branches with any review comment.
  Anything outside this narrow scope is left for human triage. The sweeper is allowed to be
  conservative — accumulating branches over the 7-day window is acceptable; over-deleting is not.

- **Stale-term dictionary:** `STALE_TERMS: dict[str, str]` at module top of
  `reflections/docs_auditor.py`. Seeded with 4 existing terms from `run_feature_docs_audit`.
  Extensible by editing one file.

- **Rotation state:** Single Redis hash `docs_audit:last_run` with fields keyed by `{path_slug}`
  for repo docs and `vault:{project_key}:{path_slug}` for vault docs. Values are float Unix
  timestamps. See the Data Flow Implementation Note for rationale (avoids per-file key explosion).

- **Zero-diff gate:** `git diff --quiet` check before `gh pr create`. If no diff, update rotation
  state and return — no empty PR created.

- **PR mode:** Reflection always opens **non-draft** PRs. No `DRAFT_MODE` constant, no flip step,
  no follow-up commit needed. The seesaw mitigation does not depend on draftness — see Risk 1.

### Flow

**Reflection scheduler fires** → picks least-recently-audited primary doc → acquires SETNX lock
→ auth probes → neighborhood resolution (≤20 files) → auto-fix detectors → file-as-issue detectors
→ zero-diff gate → push branch + open PR → Telegram notification → update rotation state

**`/do-docs` invoked** → sub-agents A–D run → substrate with `scope_mode=pr-changed-files`
→ fixes applied to current branch → commit → stage marker written → plan frontmatter updated

### Technical Approach

**Module placement:** `reflections/docs_auditor.py` (new file). The existing `scripts/docs_auditor.py`
stays alive during the ramp period only.

**Ramp period coexistence (3 phases):**
- **Phase 1:** `docs-auditor` reflection exists but `enabled: false`. Old `documentation-audit`
  and `feature-docs-audit` still run. New substrate written and unit-tested.
- **Phase 2:** Enable `docs-auditor`. **Disable (set `enabled: false`)** `documentation-audit`
  and `feature-docs-audit` in `config/reflections.yaml` in the **same commit** that enables
  `docs-auditor`. The disabled reflections do not run, so seesaw is impossible — it is single-writer
  by yaml flag, not by draftness.

  **Implementation Note (O1 — Phase 2 liveness signal):** Each daily run writes a
  `docs_audit:last_completed_run_ts` Redis key (float Unix timestamp) and an
  `docs_audit:last_completed_run_summary` Redis key (short JSON: `{"slug": ..., "pr_url": ...,
  "files_touched": N, "status": "ok"|"skipped"|"error"}`). The PM can `redis-cli GET` either key
  to confirm liveness without parsing logs. Empty PR URL with `status="skipped"` is the expected
  zero-diff signal.

- **Phase 3:** Gated on the criteria below. Delete old callables, reflections, and skill dirs.
  `knowledge-reindex` deleted in Phase 1 (it's broken and has no ramp concern).

  **Implementation Note (O2 — Phase 3 gate criteria):** Phase 3 begins only when ALL of:
  (a) at least 3 successful daily runs visible via `docs_audit:last_completed_run_ts` (no
  exceptions for 3 consecutive days); (b) at least 1 of those runs has produced a non-zero-diff
  PR (proves the auto-fix path actually works); (c) zero new GitHub issues filed against the
  `docs-auditor` reflection during the 3-day window. If (b) is missing after 7 days, the PM
  manually invokes `/do-docs` (substrate caller) once to exercise the apply path, then proceeds.

**`git log --follow` for rename detection:**
```python
result = subprocess.run(
    ["git", "log", "--follow", "--diff-filter=R", "--name-status", "--format=", "--", str(old_path)],
    capture_output=True, text=True, cwd=repo_root
)
# Parse "R100\told_name\tnew_name" lines
```
Cap at 10 queries per run to avoid slow performance on large repos. If cap exceeded, skip rename
detection for remaining files (log warning, treat as no rename).

**Telegram notification:** Mirrors `_send_log_review_telegram` exactly: same swallow-and-warn
pattern, same `timeout=10, check=False, capture_output=True`. Target: `Dev: Valor`.

**Branch naming:** `docs-audit/{doc-slug}-{ts}` where `ts` = `datetime.now().strftime("%Y%m%d-%H%M")`.

**File-as-issue deduplication:** Redis set `docs_audit:issues_filed:{title_hash}` prevents
re-filing the same issue across multiple daily runs.

**`scripts/docs_auditor.py` fate:** Delete after Phase 3 validation. No callers remain once
`run_documentation_audit` is gone.

## Risks

### Risk 1: Seesaw during ramp period
**Impact:** New reflection and old `documentation-audit` edit the same file.
**Mitigation:** Single-writer by yaml flag. The Phase 2 commit atomically flips `docs-auditor`
to `enabled: true` and both `documentation-audit` and `feature-docs-audit` to `enabled: false` in
the same `config/reflections.yaml` change. Disabled reflections do not run, so the new reflection
is the only writer at any given time. No reliance on draftness — the new reflection always opens
non-draft PRs to a separate `docs-audit/*` branch, which never conflicts with the old reflection's
direct commits to main even if the yaml change were partially applied.

### Risk 2: SDLC regression in `/do-docs`
**Impact:** In-flight PR in DOCS stage calls refactored `/do-docs`; substrate bug stalls pipeline.
**Mitigation:** `/do-docs` falls back to its existing four-agent approach if `audit()` raises.
Substrate is integration-tested before the `/do-docs` refactor lands.

### Risk 3: `git log --follow` performance
**Impact:** Rename detection for a large neighborhood takes minutes.
**Mitigation:** Cap `git log --follow` queries at 10 per run. Remaining files: skip rename
detection (log warning, treat as no rename).

### Risk 4: Stale `docs-audit/*` PRs accumulate
**Impact:** `docs-audit/*` PRs pile up if nobody reviews them.
**Mitigation:** `do-docs-branch-sweeper` reflection deletes `docs-audit/*` branches older than
7 days with no open PR; closes open `docs-audit/*` PRs older than 14 days via `gh pr close`.

## Race Conditions

### Race 1: Two `docs-auditor` reflection instances run concurrently
**Location:** `reflections/docs_auditor.py` top of callable
**Trigger:** Worker restart during reflection execution spawns two scheduler loops temporarily.
**Data prerequisite:** SETNX lock must be acquired before any file is touched.
**State prerequisite:** Lock check before neighborhood resolution.
**Mitigation:** `r.set("docs_audit:running:{slug}", "1", nx=True, ex=3600)` at callable start.
Second caller gets `None` from `r.set()`, returns `{"status": "skipped", "reason": "locked"}`.

**Implementation Note (AD1 — lock acquisition order):** The lock is acquired **before** rotation
selection, **after** auth probe, with a global key `docs_audit:running:global` (not per-slug)
because the rotation pick itself is the contended resource. Sequence:
(1) auth probe (cheap, no side effects) → (2) `r.set("docs_audit:running:global", "1", nx=True,
ex=3600)` → (3) on failure, return `{"status": "skipped", "reason": "locked"}` → (4) on success,
read `docs_audit:last_run:*` keys and select primary doc → (5) work → (6) `try/finally` ensures
`r.delete("docs_audit:running:global")` on exit so the next scheduled run does not have to wait
out the 1h TTL. The TTL is a safety backstop only.

### Race 2: Two `docs-audit/*` PRs for the same primary doc
**Location:** `reflections/docs_auditor.py::_push_and_pr`
**Trigger:** Daily reflection runs while human is reviewing a previous PR for the same primary doc.
**Data prerequisite:** None — `git push` succeeds to a new timestamp branch.
**Mitigation:** Timestamp in branch name prevents collision. Two open PRs for the same doc is
acceptable. The sweeper handles stale ones.

**Implementation Note (AD3 — branch sweep specifics):** `do-docs-branch-sweeper` enumerates
`docs-audit/*` branches via `gh api repos/{owner}/{repo}/git/refs/heads/docs-audit` (paginated).
For each branch, it queries `gh pr list --head <branch> --state all --json number,state,createdAt`.
Disposition: (a) no PR ever opened AND branch older than 7 days → `git push origin --delete <branch>`;
(b) open PR older than 14 days → `gh pr close --delete-branch <number>`; (c) merged PR → leave
alone (GitHub auto-deletes). The sweeper acquires its own lock `docs_audit:sweeper:running` (same
SETNX pattern, 30min TTL) so it cannot race with the auditor's branch push. It runs at a different
schedule slot from the auditor in `config/reflections.yaml` (offset by ≥1h). The sweeper never
touches `session/*` branches — that remains `agent/session_revival.py:204`'s scope.
