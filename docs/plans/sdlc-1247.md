# Consolidate docs hygiene into a unified auditor substrate

## Problem

Documentation hygiene in this repo is split across five disjointed pieces with no shared logic, no shared output format, and inconsistent safety properties. Three are reflections (`feature-docs-audit`, `documentation-audit`, `knowledge-reindex`) and two are skills (`/do-xref-audit`, `/do-docs-audit`). Plus the SDLC stage `/do-docs` reimplements its own version of doc cascade. They drift apart, occasionally fight each other (SDLC `/do-docs` and reflection `documentation-audit` can edit the same files with different rules), and one (`knowledge-reindex`) has been silently broken in production for a week.

**Current behavior:**

The five pieces produce inconsistent outputs (some commit to main, some open PRs, some only log), use different detector logic (4 hardcoded stale terms vs. LLM judgment), and create maintenance burden. There is no rotation — every reflection re-audits its full scope every day. There is no broken-internal-link detection anywhere. The broken `knowledge-reindex` reflection raises `ImportError: cannot import name 'reindex_vault'` on every daily run since #1167 (2026-04-26).

**Desired outcome:**

One auditor substrate (a Python module), used by two callers:
1. A new daily reflection that picks one primary doc via least-recently-audited rotation, fans out to its neighborhood (≤20 files), applies safe transforms, files issues for judgment-required findings, and pushes results as a draft PR with Telegram notification.
2. The existing `/do-docs` SDLC stage, refactored as a thin caller of the same substrate scoped to PR-changed files.

`/do-xref-audit` and `/do-docs-audit` skills are deleted (their behavior absorbed). `knowledge-reindex` reflection is deleted outright (broken + redundant with `bridge/knowledge_watcher.py`).

## Decisions

The issue's Downstream section called out 6 decisions the plan must make. Each is resolved here.

### Decision 1: Rotation state mechanism — **JSONL file** (`data/docs-audit-history.jsonl`)

**Choice:** File-based JSONL append log, one record per run.

**Why:** Aligns with the existing `daily-integration-audit` precedent (`data/audit-history.jsonl`). Survives Redis outages cleanly (the audit history stays intact even if Redis goes down). Auditable — `git log -p data/docs-audit-history.jsonl` shows historical audit decisions over time. Cross-machine consistency comes from the file being committed to the repo.

**Why not Redis hash:** No real benefit. Reflections take a Redis SETNX lock anyway (Decision below), so Redis is already required at runtime. But durable rotation state in Redis would be machine-local (each machine's Redis is separate), which is wrong — we want a single canonical audit history across machines.

**Schema:**
```jsonl
{"primary": "docs/features/foo.md", "ts": 1714579200, "scope_mode": "rotate", "neighborhood": ["docs/features/foo.md", "docs/features/bar.md"], "fixes_applied": 3, "issues_filed": 1, "branch": "docs-audit/foo-1714579200", "pr_url": "https://github.com/.../pull/N", "skipped": false}
```

Last-audit-ts per file is computed on read by max(ts) over all records where `path in record.neighborhood`. The "least-recently-audited primary" is computed by reading the union of all `.md` files under the configured roots, joining with the JSONL's reduced view, and picking the file with the oldest computed ts (or never-audited).

### Decision 2: Zero-diff PR cleanup — **Both** (pre-commit gate + sibling sweeper)

**Pre-commit gate (inside auditor):** After running detectors and applying fixes, run `git diff --quiet --cached`. If exit code 0 (no staged changes), skip the commit, skip the branch push, skip the PR. Log "audit complete, no findings." This handles the "auditor saw nothing to do" case — 90% of expected runs.

**Sibling sweeper (new sub-reflection `docs-audit-pr-cleanup`):** Daily reflection that:
1. Lists open `docs-audit/*` PRs via `gh pr list --search "head:docs-audit/" --state open --json number,headRefName,createdAt`
2. For each PR, checks `gh pr diff <N> | wc -l`
3. If diff is empty, closes the PR with `gh pr close <N> --comment "Closing — diff is empty (likely fixed via another path)"`
4. Deletes the local + remote branch via `git push origin --delete <branch>` and `git branch -D <branch>` (only if no commits ahead of main)

This handles the "auditor's fix landed elsewhere between commit and review" case.

**Why both:** The pre-commit gate catches the common case cheaply. The sweeper catches the rarer case where redundancy emerges after the PR is open. Neither alone is sufficient.

### Decision 3: Stale-term dictionary location — `config/docs_auditor_terms.yaml`

**Choice:** YAML file alongside `config/reflections.yaml`.

**Why YAML:** Aligns with existing config patterns (`reflections.yaml`). Easier for humans to PR-edit than embedded Python literals. Schema is small and stable.

**Schema:**
```yaml
# Stale-term renames detected by the docs auditor.
# When a doc references {old_term}, the auditor replaces it with {new_term}.
# Only listed terms are auto-fixed; everything else is judgment-mode.
renames:
  SessionLog: AgentSession
  RedisJob: AgentSession
  session_log: agent_session
  redis_job: agent_session
  # ... add more as renames happen across the codebase

# Terms whose presence in a doc is a finding but no auto-fix is possible
# (no canonical replacement). Auditor files an issue for each occurrence.
deprecated:
  HardCodedClient:
    reason: "Replaced by client registry; no automatic mapping — file as issue"
```

The dictionary loads on module import. Adding a new rename = PR to `config/docs_auditor_terms.yaml`. The 4 existing hardcoded terms in `reflections/auditing.py:635-640` migrate to the YAML on day one.

### Decision 4: Telegram notification chat — `Dev: Valor`

**Choice:** Same as `daily-log-review` — the engineering channel.

**Why:** This is engineering output (audit summaries, PR URLs). No CEO-facing concerns. `daily-log-review` already posts there and the pattern is a copy of `_send_log_review_telegram` in `reflections/auditing.py:240-299`.

**Message format (per run with findings):**
```
📝 docs-auditor — sdlc-1247-foo
Primary: docs/features/foo.md (last audited 14d ago)
Neighborhood: 12 files
Fixes applied: 3 stale references, 1 broken link
Issues filed: 1 (stub doc detected)

```

Skip the message entirely if pre-commit gate fired (no findings = no notification, avoid noise).

### Decision 5: Deletion sequence — staged across 3 phases

The old pieces stay running through Phase A so the new auditor proves itself before old coverage goes away. Critical because `documentation-audit` currently does mutate docs (commits to main), and we cannot leave a coverage gap while the new substrate ramps.

**Phase A (build + dry-run, ~14 days after substrate ships):**
- New substrate built, registered as `docs-auditor` in `config/reflections.yaml` with `enabled: false` initially
- Manually invoke a few times via `python -c "from reflections.docs_auditor import audit; audit('docs/features/foo.md')"` to validate behavior
- Flip to `enabled: true` with `--draft` PRs only (default)
- Old pieces still running concurrently (no behavior change for users)
- Acceptance gate: 5+ docs-audit draft PRs reviewed and merged successfully; zero false-positive autofixes

**Phase B (cutover, after Phase A acceptance):**
1. Delete `feature-docs-audit` reflection (low risk — only logs findings; new auditor has superset coverage)
2. Delete `knowledge-reindex` reflection (broken anyway; cite explicit confirmation that `bridge/knowledge_watcher.py` is the live indexer path)
3. Disable `documentation-audit` reflection (`enabled: false`) for 7 days. If no regressions, delete.
4. Refactor `/do-docs` skill body into thin caller. Run 3+ real SDLC pipelines with the new code path.
5. Promote `docs-auditor` from `--draft` to non-draft (if Phase A and SDLC pipelines pass cleanly)

**Phase C (cleanup, after Phase B settles):**
1. Delete `/do-xref-audit/` skill directory
2. Delete `/do-docs-audit/` skill directory
3. Delete `run_feature_docs_audit`, `run_documentation_audit`, `run_knowledge_reindex` from `reflections/auditing.py` and `reflections/memory_management.py`
4. Update `docs/features/reflections.md` to remove references to deleted reflections; add `docs/features/docs-auditor.md`
5. Final test run: `python -m pytest tests/`, `/sdlc` integration test on a real PR

**Critical rule:** If at any phase a regression appears, halt advancement and fix before continuing. The phases are independent — a Phase A failure doesn't roll back Phase 0 (substrate exists, just not enabled).

### Decision 6: `/do-docs` migration testing strategy

`/do-docs` is load-bearing in the SDLC pipeline. The thin-caller refactor must not break in-flight PRs. Strategy:

1. **Build the substrate first**, leave `/do-docs` skill body untouched. The substrate exists alongside the old logic.
2. **Add an internal flag** in the `/do-docs` skill body: a section header "Use new substrate" that, when present, causes the skill to call the substrate; absence preserves the old logic. Default: present (use new substrate). This is a soft gate, not a feature flag — the flag goes away in Phase C cleanup.
3. **Test on a real SDLC pipeline** before committing to the cutover. Spin up an SDLC issue with `/sdlc`, drive it to the DOCS stage, observe:
   - Stage marker `DOCS in_progress` written ✓
   - Substrate runs with `--scope=pr-changed-files`
   - Commits land on the PR branch (not new branch) ✓
   - Plan frontmatter `status: docs_complete` written ✓
   - Stage marker `DOCS completed` written ✓
   - SDLC router transitions to MERGE
4. **Run on 3+ real PRs** before promoting to default and removing the soft gate.
5. **Acceptance:** new and old behavior produce equivalent commits when run against the same PR (compared via `git diff` of the resulting commits).

## Solution

### Key Elements

- **DocsAuditorSubstrate** (new module — likely `reflections/docs_auditor/__init__.py` with submodules `detectors.py`, `fanout.py`, `state.py`, `delivery.py`): The shared substrate. Public callable `audit(primary_path, *, scope_mode, apply_mode, project_key)` returns a structured result.
- **`docs-auditor` reflection** (new entry in `config/reflections.yaml`, callable: `reflections.docs_auditor.run_rotation`): Daily, picks primary via least-recently-audited, calls `audit(...)` with `scope_mode=rotate`, `apply_mode=apply`, opens draft PR.
- **`docs-audit-pr-cleanup` reflection** (new entry in `config/reflections.yaml`, callable: `reflections.docs_auditor.run_pr_cleanup`): Daily sweeper for stale/empty docs-audit PRs.
- **`/do-docs` thin caller** (refactored skill body): Calls substrate with `scope_mode=pr-changed-files`, `apply_mode=apply`, commits to current PR branch.
- **`config/docs_auditor_terms.yaml`** (new config file): Stale-term rename dictionary.
- **`data/docs-audit-history.jsonl`** (new state file): Rotation history (gitignored if private; see Open Question).

### Flow

**Rotation reflection (Caller A) flow:**

`reflection_scheduler tick (daily)` → `run_rotation()` reads `data/docs-audit-history.jsonl` + walks all configured roots → picks least-recently-audited primary → acquires Redis SETNX lock `docs_audit:running:{slug}` → calls `audit(primary, scope_mode=rotate, apply_mode=apply, project_key=...)` → substrate computes neighborhood (transitive fanout, ≤20) → runs detectors → applies safe fixes → if zero changes: log "no findings", release lock, exit (pre-commit gate) → else: stage changes, create branch `docs-audit/{slug}-{ts}`, commit, push, `gh pr create --draft`, post Telegram message → append record to `data/docs-audit-history.jsonl` → release lock.

**SDLC `/do-docs` (Caller B) flow:**

`/do-build` invokes `/do-docs` after PR push → skill body writes stage marker `DOCS in_progress` → calls `audit(pr_changed_files[0], scope_mode=pr-changed-files, apply_mode=apply, project_key=...)` for each changed doc-relevant file → substrate runs detectors with neighborhood scoped to PR-changed files only → applies fixes inline on current PR branch (no new branch) → stage updates plan frontmatter `status: docs_complete` → stage marker `DOCS completed` → SDLC router transitions to MERGE.

### Technical Approach

**1. Substrate API (final).** Exposed at `reflections.docs_auditor.audit`:

```python
def audit(
    primary_path: Path,
    *,
    scope_mode: Literal["rotate", "pr-changed-files", "manual"],
    apply_mode: Literal["apply", "report-only"],
    project_key: str,
    commit_target: Literal["new-branch", "current-branch"] = "new-branch",
) -> AuditResult:
    """Run the unified docs auditor on `primary_path` and its neighborhood.

    Returns AuditResult with:
        fixes_applied: list[FixRecord]  # what was auto-edited
        issues_filed: list[str]         # GitHub issue URLs (judgment cases)
        branch: str | None              # branch pushed (None if pre-commit gate fired)
        pr_url: str | None              # draft PR URL (None if no commit)
        skipped: bool                   # true if pre-commit gate fired or auth missing
        skip_reason: str | None
    """
```

**2. Detector classes.** Two registries inside `reflections/docs_auditor/detectors.py`:

```python
AUTOFIX_DETECTORS = [
    detect_renamed_internal_links,    # via git log --follow --diff-filter=R
    detect_renamed_code_refs,          # paths/symbols renamed in codebase
    detect_dead_index_entries,         # README index points at deleted file
    detect_stale_term_renames,         # config/docs_auditor_terms.yaml
]

ISSUE_DETECTORS = [
    detect_deleted_targets,            # link target gone, no rename
    detect_stub_docs,                  # <5 content lines
    detect_overlapping_docs,           # LLM judgment via DocsAuditor verdict
    detect_orphan_plans,               # docs/plans/*.md with no PR/issue link
    detect_mtime_drift,                # signal only, never fix
]
```

Each detector returns a `Finding` with `auto_fix: Callable | None`. Substrate iterates: if `auto_fix is not None`, call it (this is the "safe transform"); else file as issue. Hard rule: detectors with ambiguous fix punt to issue mode (`auto_fix=None`).

**3. Fanout.** `reflections/docs_auditor/fanout.py` exposes `compute_neighborhood(primary, max_files=20) -> list[Path]`:
- Outbound: parse markdown links from `primary`, follow to existing files
- Inbound: `grep -lr "primary_basename"` across `docs/`, `.claude/`, `CLAUDE.md`
- Code symbol fanout: extract code path/symbol mentions in `primary` (regex on backticks), grep for each in the codebase, include any docs that also mention them
- One more hop: re-run inbound on each first-degree neighbor
- Stop at 20 total. Log a warning if cap was hit (signals oversized neighborhood).

**4. Auth probes.** `reflections/docs_auditor/auth.py`:
- Required: `_check_anthropic_auth()` — adopts existing `scripts/docs_auditor.py:633-662` pattern verbatim. Probe via `client.models.list()`. Failure → substrate returns skipped=True with skip_reason=auth.
- Optional: `_check_embedding_auth()` — checks `OPENAI_API_KEY` then `VOYAGE_API_KEY`. If missing, sets `embeddings_available=False`; substrate proceeds with lexical-only matching (no semantic impact agent).

**5. Concurrency.** Redis SETNX in `reflections/docs_auditor/state.py`:
```python
import redis
r = redis.Redis(...)
def acquire_lock(primary_slug: str, ttl_seconds: int = 3600) -> bool:
    return r.set(f"docs_audit:running:{primary_slug}", "1", nx=True, ex=ttl_seconds)
def release_lock(primary_slug: str) -> None:
    r.delete(f"docs_audit:running:{primary_slug}")
```
The reflection acquires before mutating, releases on success or in `finally`. TTL acts as a stuck-process safety net.

**6. PR creation.** `reflections/docs_auditor/delivery.py::open_draft_pr(branch, body) -> pr_url`:
```python
result = subprocess.run(
    ["gh", "pr", "create", "--draft", "--title", title, "--body", body, "--head", branch],
    capture_output=True, text=True, check=True
)
return result.stdout.strip()
```
The `--draft` flag is the dry-run mechanism. Promotion to non-draft = remove `--draft` from this single line (Phase B step 5).

**7. Vault iteration (per-project).** `run_rotation()` loads `~/Desktop/Valor/projects.json`, iterates projects with `knowledge_base` set, picks the least-recently-audited vault page across all projects (one vault page per run, alongside one project doc per run — the substrate can audit one of each per tick). Per-project rotation state lives in the same JSONL with `project_key` field.

- [ ] Test `audit(primary_path=non_existent_file)` — must return `skipped=True` with clear reason

### Error State Rendering
- [ ] If Anthropic auth fails: Telegram notification skipped (no spammy "audit failed" messages); reflection log shows skip_reason=auth
- [ ] If `gh pr create` fails: branch is pushed but PR isn't; substrate logs the gh error and returns `pr_url=None`; sibling sweeper picks up the orphaned branch on next tick
- [ ] If Redis is unreachable: substrate falls back to `runtime_lock_disabled=True`, logs warning, continues (single-run cost is acceptable; no cross-machine concurrent runs in practice — only one machine has the reflection enabled)

## Rabbit Holes

- **Generalizing "stale code reference" detection beyond renames.** Tempting to detect every dead path/symbol mention. But "dead reference with no rename" is judgment-mode (file as issue), not auto-fix. The detector dictionary stays bounded.
- **Markdown formatting/linting integration.** Tempting to fold `markdownlint --fix` in. But that's a different concern (formatting vs. semantic staleness) and adds a Node.js dependency. Keep separate.
- **Embedding-based duplicate doc detection.** The "overlapping docs" detector could in principle compare embeddings between docs in the same neighborhood. But Phase 0.7 research surfaced no battle-tested pattern, and this slips into a research project. Use LLM judgment via the existing `DocsAuditor` verdict instead.
- **Vault organization (file moves inside `~/work-vault/`).** Out of scope. Vault organization is a human activity. The auditor only audits, never reorganizes.
- **Replacing `bridge/knowledge_watcher.py`.** The watcher is the canonical real-time vault-to-KnowledgeDocument sync. The auditor's vault rotation is additive (audits one vault page for staleness/xref); it does not touch indexing. Don't conflate the two.
- **DocumentChunk dead code.** Recon found `DocumentChunk.search()` is only invoked in tests. Tempting to delete. Out of scope — separate cleanup, may matter for future RAG.

## Risks

### Risk 1: Auditor's autofix produces a wrong "fix" that lands in a draft PR and gets merged

**Impact:** Documentation drift introduced by the very tool meant to prevent drift.

**Mitigation:**
- Hard rule: only auto-fix when the rename is precisely identifiable via `git log --follow --diff-filter=R` (50% similarity index — git's documented default) or via the explicit dictionary in `config/docs_auditor_terms.yaml`
- All detectors have unit tests with rename/non-rename fixtures
- The 14-day Phase A draft-PR ramp gives reviewers time to catch any false positives before promotion
- Each draft PR is human-reviewed before merge during Phase A; after Phase A, draft PRs are still reviewed but at lower per-PR scrutiny

### Risk 2: `documentation-audit` removal in Phase B leaves a coverage gap

**Impact:** Some staleness check stops running, drift accumulates.

**Mitigation:**
- Phase A acceptance gate requires the new auditor to demonstrate detection parity (review 5+ draft PRs, confirm they catch issues `documentation-audit` would have flagged)
- Phase B disables `documentation-audit` for 7 days before deletion. If a regression is detected during that window, re-enable trivially via YAML flag.
- Keep `documentation-audit` code in git history; revert is a single `git revert` away

### Risk 3: `/do-docs` thin-caller refactor breaks an in-flight SDLC pipeline

**Impact:** A PR's DOCS stage fails, the SDLC router can't transition to MERGE, the PR is stuck.

**Mitigation:**
- Soft gate ("Use new substrate" header in the skill body) lets builders fall back to old logic mid-flight
- 3+ real SDLC pipelines tested before promoting the new code path to default
- The old skill body stays in git history; if a pipeline gets stuck, the operator can manually invoke `/sdlc` with the old `/do-docs` body via revert
- Acceptance criterion: no in-flight PR's DOCS stage is interrupted during cutover

### Risk 4: JSONL rotation file grows unbounded

**Impact:** `data/docs-audit-history.jsonl` grows by 1 record/day per primary. Over a year, that's ~365 records — small. But across years and per-project growth, eventually slow to read.

**Mitigation:**
- Truncate on read: only keep the latest record per (path, project_key) tuple. The rotation logic only needs the latest per primary anyway.
- Add a sibling cleanup task in the existing `analytics-rollup` reflection (or a new tiny one) to compact monthly: read all records, keep latest per primary, rewrite file. Trivial Python.
- File-size monitoring via `disk-space-check` reflection picks up runaway growth.

## Race Conditions

### Race 1: Two reflection runs picking the same primary
**Location:** `reflections/docs_auditor/state.py::run_rotation`
**Trigger:** Two machines or two ticks within the same machine race to acquire rotation slot
**Data prerequisite:** `data/docs-audit-history.jsonl` reflects current state
**State prerequisite:** Redis lock not held by another process
**Mitigation:** Redis SETNX `docs_audit:running:{primary_slug}` with 1h TTL acquired before any mutation. If lock acquisition fails, reflection logs "audit already in progress, skipping" and exits cleanly. The TTL is a backstop against crashed processes.

### Race 2: Auditor commits while a human is editing the same file
**Location:** `reflections/docs_auditor/__init__.py::audit`
**Trigger:** Concurrent edit on a feature branch and the auditor's branch
**Data prerequisite:** Both edits are pushed to origin
**State prerequisite:** Branches are independent; merge happens at PR review
**Mitigation:** No mitigation needed at the auditor level — git's standard merge resolution at PR review handles conflicts. The auditor never edits main directly; all edits go through draft PR. Conflict = reviewer rebases or closes.

### Race 3: SDLC `/do-docs` and rotation reflection both target the same PR's docs
**Location:** `reflections/docs_auditor/__init__.py::audit` (both callers)
**Trigger:** A docs-related PR is in flight; rotation tick happens to pick a doc that's also in the PR
**Data prerequisite:** PR is open, doc is part of both scopes
**State prerequisite:** PR has not yet merged
**Mitigation:** Rotation reflection's primary-picker excludes any path that is currently part of an open `docs-audit/*` or `session/*` PR's diff. A small read-only `gh pr list --state open --json number,files` filter at primary-pick time. Cheap (one gh call per tick) and prevents the race entirely.
