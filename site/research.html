06 · Strategic research

Reading the agent landscape

Valor studies the platforms and repos shipping around it, and each study asks a version of the same question: which layer of the stack is worth owning, and which is better rented, adopted, or simply watched. Below: the watchlist of systems worth learning from, and the comparison studies written along the way.

01 · The watchlist

Systems worth learning from

These are the inspirations and rivals that shaped design decisions — each entry is what the system is, and what Valor took from studying it.

Claude Managed AgentsAnthropic · hosted harness
A pre-built agent loop in managed cloud containers, driven over an API. The designated future "hands" of the system — execution offload waits on hosted memory and a hook-equivalent API.
Paperclipopen source · control plane
Org charts, per-agent budgets, and approval gates over any agents. Source of the budget hard-stop, goal-aligned tracing, and immutable audit-log ideas being adopted natively.
HermesNous Research · headless framework
A headless multi-tenant agent across twenty chat platforms, with agent-curated memory and self-improving skills. The strongest argument for indexed session stores — and the skills ecosystem worth watching.
OpenHumantinyhumans · desktop app
A one-user second brain that normalizes everything into readable Markdown trees. The strongest argument for a legible file store — half of why Valor's memory runs two stores.
Perplexity ComputerPerplexity · digital worker
A vision-language agent that drives a real browser like a person. The benchmark for visual web work — and a lesson in why an orchestrator needs an API, not a coworker.
Pi & Gondolinminimal core · micro-VM sandbox
A widely-adopted minimal agent core and a standalone micro-VM sandbox with the cleanest isolation seam evaluated — deliberately not adopted, because Valor's dedicated machine is its isolation boundary.
OmnigentDatabricks · control plane
A credible open-source agent control plane that looked like a toy in second-hand accounts and wasn't — the standing reminder to verify maturity claims at the source.
jcodecoding-agent runtime
A fast-moving single-author coding runtime: high capability, no integration seam. The pattern for when to watch rather than adopt.

02 · The comparisons

Head-to-head studies

Full write-ups of the in-house strategic analyses, one page per matchup.

April 2026 Valor vs Paperclip A vertical teammate meets a horizontal control plane. Complementary, not competitive — and the best ideas (budget hard-stops, goal tracing, audit logs) are worth adopting natively before adopting the platform. April 2026 Valor vs Managed Agents Valor orchestrates, Managed Agents executes. The hard problem is porting subconscious memory to a hosted harness — solved by stealth injection through workflow tools, with shared Redis as the linchpin. April 2026 Managed Agents vs Perplexity Computer A programmable engine versus a digital worker — a fork, not a spectrum. You cannot orchestrate what has no API, and the credit economics settle whatever the API question doesn't. May 2026 OpenHuman vs Hermes Two open agent stacks, two bets: transparency for one desktop user, or autonomy and reach for a headless fleet. Memory shape follows form factor — the lesson behind Valor's own two-store design.
